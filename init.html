Документация

<p> — абзац текста
<a> — ссылка: <a href="url">текст ссылки</a>
<img> — картинка: <img src="url" />
<div> — блочный контейнер (содержимое как будто запаковывается в «коробку», по умолчанию во всю ширину экрана)
<span> — строчный контейнер (нужен для добавления форматирования тексту: «коробки» вокруг нет)
h1,h2, ... h6 — заголовки
<br> — разрыв строки

У тэгов бывают атрибуты (они пишутся после имени тэга в форме аттрибут="значение"). Пару атрибутов выше вы уже видели: у тэгов a, img. Если тэг парный, атрибуты указываются только в открывающем тэге, но не в закрывающем.
Два главных атрибута, которые есть у каждого тэга, нужны для того, чтобы было возможно сослаться на конкретный тэг или группу тэгов.

Атрибут id указывает уникальное имя тэга (оно не должно повторяться нигде на странице):
<h3 id="html_basics">Основы html</h3>

Атрибут class указывает, что тэг принадлежит конкретной группе. Например,
<span class="first-letter">Э</span>то <span class="first-letter">З</span>аглавные <span class="first-letter">Б</span>уквы
Тут три буквы «завёрнуты» в тэги span (строчный контейнер), и каждому из этих тэгов добавлен класс first-letter (да, названия через дефис разрешены). Теперь мы можем обращаться к этим трём буквам как к группе. Например, для того чтобы стилизовать (см. css).

———

Вообще страница состоит из четырёх основных типов элементов:
* html — основное содержимое страницы и базовая структура.
* css — стили. Т.е. оформление страницы (как элементы страницы расположены друг относительно друга, с какими отступами, какого цвета, какого размера итд)
* js — скрипты, т.е. поведение страницы
* images, fonts — картинки, шрифты итп

Браузер при первом запросе к серверу получает html-документ. Находит в нём ссылки на стили, скрипты, картинки итп. И делает новые запросы. В итоге браузер склеивает всё в единую страницу.
